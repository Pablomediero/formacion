GIT FLOW

Link

Ventajas

  • Este flujo de trabajo es ideal para los flujos de trabajo de software basados en publicaciones.
  • Gitflow ofrece un canal específico para las correcciones de producción.

Características

Utuliza 2 ramas
  • main: Historial de publicacion oficial
  • develop: Integracion para las funciones

Para crear la rama develop se utiliza

Creación
git flow init

Ramas de Función:

LLamadas ramas feature utilizan develop como rama primaria(no interactuan directamente con main), al finalizar el trabajo de desarrollo hay que fusionar la rama con develop

CreaciónFinalización
git flow feature start feature_branch git flow feature finish feature_branch

Ramas de Publicación:

Llamadas ramas release, cuando develop haya adquirido suficientes funciones para publicar, en esta rama solo deben producirse las soluciones de errores, generacion de documentacion y tareas orientadas a la Publicación cuando este lista se fusiona con main y se agrega un tag con número de versión y fusionarse con develop

CreaciónFinalización
git flow release start 0.1.0 git flow release finish '0.1.0

Rama de Correción:

LLamadas ramas hotfix son las ramas de mantenimiento, sirven para reparar rapidamente las publicaciones en producción están basadas en main(única rama a fusionar con main) y se debe marcar con un tag actualizado esto permite que el equipo solucione las incidencias sin interrumpir el flujo de trabajo

CreaciónFinalización
git flow hotfix start hotfix_branch git flow hotfix finish hotfix_branch
